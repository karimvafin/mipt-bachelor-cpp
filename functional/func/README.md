# Функции как объекты в C++

Цель: понять, как передавать и хранить функции в C++, изучить три основных инструмента — сырые указатели на функции, `std::function` и лямбда-функции — и знать, когда применять каждый из них.

## Указатели на функции (raw function pointers)

Понятия:

- Указатель на функцию — переменная, хранящая адрес функции с конкретной сигнатурой.
- Сигнатура — тип возвращаемого значения и типы параметров; указатель подходит только к функции с совпадающей сигнатурой.
- Синтаксис объявления — `ReturnType (*name)(ParamTypes...)`.
- Псевдоним типа (`using`) — упрощает объявление: `using Comparator = bool(*)(int, int)`.
- Передача в функцию — позволяет менять поведение без изменения кода алгоритма (паттерн «стратегия»).
- Ограничение — не захватывают контекст: нельзя передать лямбду с захватом или метод класса.

Пример:

```cpp
bool less_than(int a, int b) { return a < b; }

using Comparator = bool(*)(int, int);

void sort(int* arr, int n, Comparator cmp);

sort(arr, n, less_than);
```

```
g++ -std=c++20 -Wall -Wextra -o ptrs 01-function-pointers.cpp
./ptrs
```

## std::function

Понятия:

- `std::function<Signature>` — обёртка из `<functional>`, которая может хранить любой вызываемый объект с подходящей сигнатурой.
- Вызываемый объект (callable) — то, что можно вызвать через `()`: обычная функция, указатель на функцию, лямбда, функтор, `std::bind`.
- Функтор — класс с перегруженным `operator()`.
- Type erasure — механизм, позволяющий `std::function` хранить callable любого типа, скрывая его конкретный тип.
- Накладные расходы — `std::function` использует динамическую аллокацию для больших объектов и виртуальную диспетчеризацию; дороже указателя на функцию.
- `std::bind` — адаптер, фиксирующий часть аргументов функции (частичное применение).
- Пустой `std::function` — при вызове бросает `std::bad_function_call`.

Пример:

```cpp
#include <functional>

std::function<int(int, int)> op = [](int a, int b) { return a + b; };
op = std::plus<int>{};  // можно заменить другим callable
std::cout << op(3, 4);  // 7
```

Команды:

```
g++ -std=c++20 -Wall -Wextra -o stdfunc 02-std-function.cpp
./stdfunc
```

## Лямбда-функции

Понятия:

- Лямбда — анонимный функциональный объект, создаваемый выражением `[захват](параметры) -> тип { тело }`.
- Замыкание (closure) — объект, порождённый лямбдой; хранит захваченные переменные.
- Список захвата (`[]`) — определяет, какие переменные из окружающей области видимости доступны в теле.
  - `[]` — ничего не захватывается.
  - `[x]` — захват `x` по значению (копия).
  - `[&x]` — захват `x` по ссылке.
  - `[=]` — захват всего используемого по значению.
  - `[&]` — захват всего используемого по ссылке.
  - `[=, &y]` — всё по значению, кроме `y` — по ссылке.
- `mutable` — позволяет изменять захваченные по значению переменные внутри лямбды.
- Обобщённая лямбда (C++14) — параметры типа `auto`; лямбда становится шаблонной.
- `auto` для лямбды — единственный способ хранить лямбду без её типа; тип уникален для каждой лямбды.
- Немедленный вызов (IIFE) — лямбда вызывается сразу после объявления: `[](){ ... }()`.

Пример:

```cpp
int threshold = 10;
auto above = [threshold](int x) { return x > threshold; };

// Изменение захваченной переменной не меняет оригинал
threshold = 99;
above(15);  // true, threshold внутри лямбды всё ещё 10
```

Команды:

```
g++ -std=c++20 -Wall -Wextra -o lambdas 03-lambdas.cpp
./lambdas
```

## Сравнение инструментов

| Инструмент            | Хранит состояние | Накладные расходы | Захват контекста |
|-----------------------|:----------------:|:-----------------:|:----------------:|
| Указатель на функцию  | нет              | минимальные       | нет              |
| `std::function`       | да               | есть              | да               |
| Лямбда (`auto`)       | да               | минимальные       | да               |

Правило выбора:
- Если нужна максимальная производительность и состояние не требуется — указатель на функцию.
- Если нужно хранить callable в поле класса или контейнере — `std::function`.
- Если callable используется локально — лямбда с `auto`.

## Практика

Идеи для самостоятельной работы:

- Написать функцию `apply(int* arr, int n, int(*f)(int))`, применяющую `f` к каждому элементу массива.
- Переписать её, принимая `std::function<int(int)>`, и сравнить удобство использования.
- Реализовать `make_adder(int n)` — функцию, возвращающую лямбду, прибавляющую `n` к аргументу.
- Написать `compose(f, g)` — возвращает лямбду, вычисляющую `f(g(x))`.
- Проверить, что лямбда с захватом по ссылке на локальную переменную становится висячей после выхода из функции.
