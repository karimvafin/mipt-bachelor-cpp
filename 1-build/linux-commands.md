# Основные Linux-команды

Ниже — базовые команды, которые чаще всего нужны при работе с проектами на C/C++.

## Навигация по файловой системе

- `pwd` — показать текущую директорию.
  - Пример: `pwd`
- `ls` — список файлов.
  - Примеры: `ls`, `ls -lh`, `ls -a`
- `cd` — перейти в директорию.
  - Примеры: `cd ..`, `cd ~/projects`, `cd /tmp`

## Работа с файлами и папками

- `mkdir` — создать директорию.
  - Примеры: `mkdir build`, `mkdir -p build/bin`
- `touch` — создать пустой файл или обновить время.
  - Пример: `touch main.cpp`
- `cp` — копирование файлов и папок.
  - Примеры: `cp a.txt b.txt`, `cp -r src backup/`
- `mv` — перемещение/переименование.
  - Примеры: `mv old.txt new.txt`, `mv a/ b/`
- `rm` — удаление файлов и папок.
  - Примеры: `rm file.txt`, `rm -r build/`, `rm -rf tmp/`

## Просмотр содержимого

- `cat` — вывести содержимое файла.
  - Пример: `cat README.md`
- `less` — просмотр с прокруткой.
  - Пример: `less main.cpp`
- `head` — первые строки файла.
  - Пример: `head -n 20 file.txt`
- `tail` — последние строки файла.
  - Пример: `tail -n 20 file.txt`

## Поиск

- `grep` — поиск строк в тексте.
  - Примеры: `grep "main" *.cpp`, `grep -R "TODO" src/`
- `find` — поиск файлов.
  - Пример: `find src -name "*.cpp"`

## Права доступа

- `chmod` — изменить права доступа.
  - Примеры: `chmod +x app`, `chmod 644 file.txt`
- `chown` — изменить владельца (обычно нужен sudo).
  - Пример: `sudo chown user:group file.txt`

## Системные утилиты

- `which` — путь к исполняемому файлу.
  - Пример: `which g++`
- `echo` — вывести текст/переменные.
  - Примеры: `echo hello`, `echo $PATH`
- `env` — показать переменные окружения.
  - Пример: `env | grep PATH`
- `history` — история команд.
  - Пример: `history | tail -n 20`
- `man` — руководство по команде.
  - Пример: `man ls`

## Архивация

- `tar` — создание/распаковка архивов.
  - Примеры: `tar -czf backup.tar.gz dir/`, `tar -xzf backup.tar.gz`
- `zip`/`unzip` — ZIP-архивы.
  - Примеры: `zip -r project.zip project/`, `unzip project.zip`

## Работа с процессами

- `ps` — список процессов.
  - Пример: `ps aux | grep app`
- `kill` — завершить процесс по PID.
  - Пример: `kill 12345`
- `top`/`htop` — интерактивный мониторинг.
  - Пример: `top`

## Сеть

- `ping` — проверить доступность хоста.
  - Пример: `ping 8.8.8.8`
- `curl` — загрузка данных по HTTP.
  - Пример: `curl https://example.com`

## Полезные сочетания

- `command1 | command2` — конвейер, передача вывода.
- `command > file` — перенаправление вывода в файл.
- `command >> file` — дописать вывод в файл.
- `command && command2` — выполнить вторую команду, если первая успешна.
- `command || command2` — выполнить вторую команду, если первая неуспешна.
